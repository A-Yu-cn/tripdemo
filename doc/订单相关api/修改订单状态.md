
    
**简要描述：** 

- 修改订单状态接口

**请求URL：** 
- ` /indents/{indentNo} `
  
**请求方式：**
- POST 

**参数：** 

|参数名|必选|类型|说明|
|:----    |:---|:----- |-----   |
|indentNo|是|string|订单编号（pathvar）|
|type|是|int|要修改的订单状态|

**订单状态说明**    

1. 未支付(默认)
2. 已支付
3. 已完成

**限制访问资源_需要登录**

 **返回示例**

``` 
{
	"mes": "",
	"data": "修改成功",
}
```
